<Callout title="Your AI doesn't understand your codebase. Mikk fixes that." type="info">
  Mikk parses your entire project via the TypeScript Compiler API, builds a bidirectional dependency
  graph, hashes everything with Merkle-tree SHA-256, and serves it through an MCP server that
  Claude, Cursor, and VS Code Copilot call directly. **Everything stays on your machine.**
</Callout>

## What is Mikk?

LLMs write great code ŌĆö for codebases they've never seen. They don't know your module boundaries, can't trace your dependency graph, and have no idea that touching `login.ts` breaks 14 downstream functions across 3 packages.

Mikk fixes that by giving AI assistants **real architectural intelligence** ŌĆö not guesses.

## The 7-Step Pipeline

Everything happens in ~3 seconds when you run `mikk init`:

<Steps>
  <Step>
    ### Parse
    Every `.ts`, `.tsx`, `.js`, `.jsx`, `.go` file parsed via the **TypeScript Compiler API** ŌĆö real AST, not regex. Functions, classes, imports, exact line ranges.
  </Step>
  <Step>
    ### Graph
    Full dependency DAG built in two passes. Both `dependsOn` and `dependedOnBy` maps ŌĆö O(1) traversal in either direction.
  </Step>
  <Step>
    ### Cluster
    Files grouped into logical modules via greedy agglomeration ŌĆö import patterns, naming conventions, directory structure.
  </Step>
  <Step>
    ### Hash
    SHA-256 Merkle tree at every level: `function ŌåÆ file ŌåÆ module ŌåÆ root`. One hash = full drift detection.
  </Step>
  <Step>
    ### Contract
    `mikk.json` written ŌĆö module boundaries, constraint rules, and ADR decisions.
  </Step>
  <Step>
    ### Context
    `claude.md` + `AGENTS.md` generated ŌĆö tiered architecture summaries, ready for any AI assistant.
  </Step>
  <Step>
    ### Serve
    MCP server starts. 15 tools, 3 resources, millisecond response times ŌĆö all reading from `mikk.lock.json`.
  </Step>
</Steps>

## Package Ecosystem

Mikk is a Turborepo monorepo with 8 npm packages under `@getmikk`:

| Package | Role |
|---------|------|
| `@getmikk/core` | AST parsing, dependency graph, Merkle hashing, contract management |
| `@getmikk/cli` | 15+ commands ŌĆö init, analyze, diff, watch, contracts, context, intent |
| `@getmikk/ai-context` | BFS graph tracing, token budgeting, `claude.md` generation |
| `@getmikk/mcp-server` | MCP server ŌĆö 15 tools, 3 resources for AI assistants |
| `@getmikk/intent-engine` | NL prompt parsing, conflict detection (6 rule types) |
| `@getmikk/watcher` | Debounced incremental watcher, atomic updates |
| `@getmikk/diagram-generator` | Mermaid.js diagrams, 8 diagram types |
| `@getmikk/vscode-extension` | VS Code tree, impact analysis, AI context, status bar |
