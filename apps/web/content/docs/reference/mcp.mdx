<Callout title="Every tool reads from mikk.lock.json" type="info">
  No re-parsing on every call. Response times are in milliseconds because the entire dependency
  graph, Merkle tree, and function index are pre-computed and stored.
</Callout>

## Quick Start

```bash
mikk mcp                          # start the server
mikk mcp install                  # auto-detect and install into Claude/Cursor/VS Code
mikk mcp install --tool claude    # install into Claude Desktop specifically
```

## Connecting AI Assistants

<Tabs items={["Claude Desktop", "Cursor", "VS Code"]}>
  <Tab value="Claude Desktop">
    Add to `%APPDATA%\Claude\claude_desktop_config.json` (Windows) or
    `~/Library/Application Support/Claude/claude_desktop_config.json` (macOS):

    ```json title="claude_desktop_config.json"
    {
      "mcpServers": {
        "mikk": {
          "command": "npx",
          "args": ["-y", "@getmikk/mcp-server", "/absolute/path/to/your/project"]
        }
      }
    }
    ```
  </Tab>

## Resources

| Resource | URI | Description |
|----------|-----|-------------|
| Lock file | `mikk://lock` | Full `mikk.lock.json` as structured JSON |
| Architecture | `mikk://architecture` | Current `mikk.json` contract |
| Context | `mikk://context` | Current `claude.md` content |

## Recommended Workflow

For best results, start every AI session with a context handoff:

```
You: "Read mikk://context to understand this codebase before we start."

Then: "I want to add rate limiting to the auth module.
       Use mikk_check_intent to validate this first,
       then mikk_get_module_detail on 'auth' to understand the current structure."
```

<Callout title="This gives your AI real architectural grounding" type="success">
  Module boundaries, existing constraints, exact function locations — all before generating any code.
  No more hallucinated import paths or nonexistent function names.
</Callout>
